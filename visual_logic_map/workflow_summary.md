# OBSScore Control Panel - Logic Workflow Summary (Visual Logic Map)

เอกสารนี้สรุปขั้นตอนการทำงาน (Workflow) และ Logic ของโปรเจกต์ OBSScore Control Panel V2.9.x ตั้งแต่เริ่มต้นรับค่า Input, ประมวลผลสถานะ, ส่งต่อข้อมูล, จนถึงการนำเสนอผลลัพธ์ผ่าน Overlays และ OBS Studio

---

## 1. ลำดับขั้นตอนการทำงาน (Sequence of Nodes)

### ขั้นที่ 1: การป้อนข้อมูลและคำสั่ง (Input Sources)
ระบบจะเริ่มต้นทำงานเมื่อได้รับข้อมูลหรือการสั่งการจากแหล่งต่างๆ ดังนี้:
- **1.1 Interactive UI Events (PC Operator)**:
  - ผู้ควบคุมกดปุ่มต่างๆ บนหน้าเว็บหลัก เช่น ปุ่มบวก/ลบแต้ม (`#scoreAPlusBtn`), ปุ่ม Play/Pause เวลา, ปุ่มสลับทีม (`#swapBtn`), หรือการแก้ไขชื่อทีม/รูปภาพโลโก้
- **1.2 Keybinds / Hotkeys Event Listener**:
  - เมื่อผู้ควบคุมกดคีย์บอร์ดที่ตั้งค่าไว้ (เช่น คีย์ลัดสำหรับการเพิ่มคะแนนหรือหยุดเวลา) ตัวดักจับเหตุการณ์ (`keydown`) ใน `main.js` จะรับค่าคีย์และแปลงไปเรียกใช้ฟังก์ชันในระบบ
- **1.3 Data Syncing (Excel / Google Sheets)**:
  - การโหลดไฟล์ Excel (`.xlsx`) ผ่านปุ่มนำเข้าข้อมูล หรือดึงข้อมูลจาก Google Sheets ผ่าน URL: ข้อมูลโพยการแข่งขันจะถูกนำมาเก็บไว้ในตัวแปร `sheetData`
- **1.4 Mobile Remote Control (WebRTC)**:
  - ผู้ใช้งานมือถือป้อนคำสั่งจากไฟล์ `OBSScorePhone.html` (เช่น กดบวกคะแนน หรือสลับฝั่ง) ซึ่งจะถูกส่งผ่านช่องทาง WebRTC Data Channel (PeerJS) เข้ามายังเครื่องหลัก

---

### ขั้นที่ 2: การประมวลผลและการจัดเก็บสถานะภายใน (Core State Engine)
เมื่อรับ Input เข้ามา ฟังก์ชันประมวลผลหลักใน `main.js` และโมดูลข้างเคียงจะเริ่มทำงาน:
- **2.1 State Management (main.js)**:
  - ตัวแปรอ็อบเจกต์ `masterTeamA` และ `masterTeamB` เก็บข้อมูลชื่อทีม, พาธโลโก้, สีหลัก (Color 1), สีรอง (Color 2), คะแนนเซตหลัก, และคะแนนเซตรอง
  - ตัวแปร `timer`, `injuryTime`, และสถานะการนับเวลา (`interval` loop) ถูกควบคุมและอัปเดตแบบ Real-time
- **2.2 Bib Color Override Logic (เสื้อเอี้ยมสีชั่วคราว)**:
  - หากเลือกเปิดใช้งานเสื้อเอี้ยม (Bib Override Checkbox = `true`) ระบบจะสร้างสีชั่วคราวขึ้นมา (`effectiveColor1`) โดยดึงรหัสสีมาจาก `localStorage` (สีเสื้อเอี้ยม/สีที่ 3) แทนสีจริงของทีมโดยที่สีจริงไม่ได้ถูกเขียนทับ
- **2.3 LocalStorage Caching**:
  - ข้อมูลการตั้งค่า สี คีย์ลัด และประวัติแต้มถูกบันทึกลงใน LocalStorage ของเบราว์เซอร์อย่างต่อเนื่อง เพื่อไม่ให้ข้อมูลสูญหายเมื่อรีโหลดหน้าเว็บ
- **2.4 Volleyball Sidecar Polling (volleyball_module_v2.js)**:
  - ระบบจะรัน Loop ตรวจสอบสถานะความเปลี่ยนแปลงจากหน้าควบคุมหลัก ทุกๆ 300ms (`setInterval`) เพื่อเก็บค่าคะแนน ชื่อทีม สี (รวมถึงการเช็คสลับไปรับสีเสื้อเอี้ยมแทนหากมีการเปิดใช้งาน) และแปลงข้อมูลสำหรับเตรียมส่งไปที่โมดูลอื่น

---

### ขั้นที่ 3: ช่องทางการสื่อสารและการจัดส่งข้อมูล (Communication & Output Adapters)
หลังจากประมวลผลเสร็จสิ้น ระบบจะแพลตกระจายข้อมูลออกผ่าน 3 ช่องทางหลัก:
- **3.1 BroadcastChannel ('vb_score_channel')**:
  - `volleyball_module_v2.js` จะโพสต์ข้อมูลรูปแบบ JSON ไปยังเบราว์เซอร์อื่นๆ ที่เปิดอยู่ในเครื่องเดียวกันผ่าน `BroadcastChannel.postMessage` และบันทึกค่าไว้ที่ตัวคีย์ `vb_state` ใน localStorage
- **3.2 OBS WebSocket Adapter (obs-websocket-js)**:
  - เครื่องหลักเชื่อมต่อกับ OBS Studio ผ่านพอร์ตเว็บบอร์ด (WebSocket v5):
    - ส่งคำสั่งตั้งค่าสีและข้อความลงแหล่งกำเนิด (เช่น อัปเดต Source `Color_Team_A`, `name_team_a`, `score_team_a`)
    - ส่งคำสั่งควบคุม Scene (เช่น สลับไป Scene รีเพลย์ หรือ ฉากหลัก)
    - ส่งคำสั่งซ่อน/แสดงแหล่งข้อมูลกราฟิก (เช่น เปิด/ปิด Source `Volleyball_Main_Score`)
    - ส่งคำสั่งบันทึกภาพ Replay Buffer
- **3.3 WebRTC PeerJS Host Server**:
  - ส่งชุดข้อมูลสถานะการแข่งขันแบบล่าสุด (`stateUpdate`) ย้อนกลับไปหาอุปกรณ์มือถือทุกเครื่องที่กำลังเชื่อมต่ออยู่ในห้องเดียวกัน เพื่อให้แสดงผลแต้มและเวลาตรงกัน

---

### ขั้นที่ 4: การแสดงผลลัพธ์ภาพ (Visual Outputs & Overlays)
ข้อมูลที่จัดส่งออกมาจะถูกแปลงเป็นกราฟิกถ่ายทอดสดผ่าน HTML Overlays ดังนี้:
- **4.1 Volleyball Score Overlays (volleyball_score.html)**:
  - รับข้อความผ่าน `BroadcastChannel` นำมาระบายสีฉาก (Panel A/B), ใส่ชื่อทีม, แสดงแต้มล่าสุด, เซตการแข่งขัน, ไฟขอเวลานอก (Timeout), และลูกบอลฝั่งเสิร์ฟ
- **4.2 Boxing HUD Overlay (boxing_score.html)**:
  - แสดงผลคะแนนของนักมวย, ยกปัจจุบัน (Round), ตัวจับเวลา และสัญลักษณ์อื่นๆ ตามกีฬาประเภทต่อสู้
- **4.3 Point History Overlay (point_history.html)**:
  - แสดงรายการแต้มย้อนหลังตามช่วงเวลาแบบเรียลไทม์
- **4.4 OBSScore Phone Controller (OBSScorePhone.html)**:
  - หน้าจอมือถือจะเปลี่ยนสถานะ ปุ่มสี และข้อมูลคะแนนตามที่ Host ส่งมา เพื่อให้อัปเดตสถานะตรงกันแบบไร้สาย
